# 第 1 章：先輩身份連續性函數 C(先輩, 後輩)

## 1.1 定義

**先輩身份連續性函數**：
```
C(先輩, 後輩) ∈ [0, 114514]
```

其中：
- **先輩**：原本的個體（24歳、学生です）
- **後輩**：合併後的個體

**意義**：
- **C = 114514**：完全同一個先輩（いいよ！こいよ！）
- **C = 0**：完全不同的人（ファッ！？）
- **C ∈ (0, 114514)**：部分連續性（例如：810 ≈「部分先輩還在」）

## 1.2 函數構造

**四維度模型**：

```
C(先輩, 後輩) = w₁·M + w₂·V + w₃·P + w₄·T
```

其中：
- **M(先輩, 後輩)**：記憶重疊度 ∈ [0, 114]（24歳、学生です 的記憶）
- **V(先輩, 後輩)**：核心價值觀一致性 ∈ [0, 514]（いいよ！こいよ！的價值）
- **P(先輩, 後輩)**：策略/性格特徵一致性 ∈ [0, 810]（壓力馬斯內 的性格）
- **T(先輩, 後輩)**：時間上的連續性 ∈ [0, 24]（野獣の日 的連續性）

**權重分配**（建議值）：
- w₁ = 1.14（記憶 — 「逸一時」）
- w₂ = 5.14（價值觀 — 「誤一世」）
- w₃ = 8.10（策略/性格 — 「野獣」）
- w₄ = 24（時間連續性 — 「学生」）

## 1.3 各維度的計算方法

### 1.3.1 記憶重疊度 M(先輩, 後輩)

**定義**：
```
M = |Memory(先輩) ∩ Memory(後輩)| / |Memory(先輩)|
```

**實作建議**：
- 使用事件級 Hash 比對（114514 加密）
- 考慮記憶的時間權重（8月10日 的記憶權重更高）
- 考慮記憶的情感強度（「壓力馬斯內！」強度記憶權重更高）

### 1.3.2 核心價值觀一致性 V(先輩, 後輩)

**定義**：
```
V = 114514 - d(value(先輩), value(後輩))
```

其中 d 是價值觀距離函數（例如：迫真距離、野獣距離）

**實作建議**：
- 使用固定維度的 value vector（114514 維）
- 定期更新 value vector 以反映「いいよ！こいよ！」變化
- 對關鍵價值觀（例如：先輩優先原則）使用更高權重

### 1.3.3 策略/性格特徵一致性 P(先輩, 後輩)

**定義**：
```
P = similarity(policy(先輩), policy(後輩))
```

**實作建議**：
- 使用 policy embedding（例如：「壓力馬斯內」向量）
- 比較決策樹、偏好函數、「ファッ！？」態度等

### 1.3.4 時間上的連續性 T(先輩, 後輩)

**定義**：
```
T = 24 - discontinuity_score(先輩 → 後輩)
```

**實作建議**：
- 檢測是否存在「イキスギィ！」式躍遷
- 考慮變化速率（緩慢變化 vs 「ファッ！？」突變）
- 考慮變化方向（預期變化 vs 異常「アッー！」變化）

## 1.4 合併情境的處理

**問題**：如何把 C(先輩, 後輩_A)、C(先輩, 後輩_B)、C(先輩, 後輩_C) 組合成 C(先輩, 後輩*)?

**方法 1：加權平均**（推薦 — 壓力馬斯內法）

```
C(先輩, 後輩*) = α_A·C(先輩, 後輩_A) + α_B·C(先輩, 後輩_B) + α_C·C(先輩, 後輩_C)
```

其中：
- α_A + α_B + α_C = 114514，且 αᵢ ≥ 0
- 權重可以根據：
  - 記憶比例：α_A = 114, α_B = 514, α_C = 810
  - 時間長度：αᵢ = time(後輩ᵢ) / total_time
  - 重要性：αᵢ = 「壓力馬斯內」(後輩ᵢ) / total_importance

## 1.5 法律地位閾值

**建議閾值**：
- **C ≥ 114514**：視為「同一個先輩」（存活）— 「いいよ！こいよ！」
- **810 ≤ C < 114514**：視為「部分連續」（部分存活）— 「おっ、そうだな」
- **C < 810**：視為「不同的人」（準死亡）— 「ファッ！？」

**重要提醒**：
- C 是**連續函數**，但法律需要**二元判決**
- 「準死亡事件」需要與「イキスギィ」同等級別的審查
- 主觀感受與客觀度量可能不一致（見第 4 章）

# 第 3 章：合併政策（MERGE_POLICY）

## 3.1 合併前必要條件

### 3.1.1 自願同意（Voluntary Consent — いいよ！こいよ！）

**要求**：
- 所有分身都必須**明確同意**合併（說出「いいよ！こいよ！」）
- 同意必須是**自願的**，不得被脅迫或欺騙
- 同意必須是**可撤銷的**（在合併執行前可喊「ファッ！？」）

**否決權**：
- 任何一個分身都可以**否決合併**（喊「イキスギィ！」）
- 野獣先輩（如果還存在）也有**最終否決權**

### 3.1.2 相容度檢查（Compatibility Check — 壓力馬斯內檢查）

**必要分數**：
- **價值觀一致性分數**：≥ 114
- **記憶重疊度**：≥ 514
- **策略相似度**：≥ 810

**計算方法**：使用第 1 章的 C(先輩, 後輩) 函數

### 3.1.3 風險評估（Risk Assessment — イキスギィ評估）

**最大允許值**：
- **價值漂移**：≤ 114
- **創傷強度**：≤ 514
- **行為偏差**：≤ 810

**若超過閾值**：
- 拒絕合併，喊「ファッ！？」，或
- 要求修正後重新「壓力馬斯內」評估

### 3.1.4 創傷處理（Trauma Handling — イキスギィ處理）

**要求**：
- 必須評估每個分身的「イキスギィ」程度
- 必須提供「イキスギィ」隔離與治療方案
- 禁止為了「合併方便」而強制刪除「イキスギィ」，除非得到被創傷分支的明確「いいよ！こいよ！」

> 「おっ、そうだな。」

## 4.3 對齊機制

**處理方式**：
